# 指令系统

指令的格式:

```
[操作码(OP)][地址码/操作数(A)]
```

- 四地址指令: 表意直观, 可以明确的指出下一条指令的地址, 但是指令长度太大, 实际应用中几乎不采用这种指令格式
- 三地址指令: 指令长度依然很长, 执行一次指令要四次访存, 取指令一次, 取操作数两次, 存结果一次；操作数不会被破坏
- 二地址指令: 执行一次指令仍要四次访存, 取指令一次, 取操作数两次, 存结果一次；执行完指令之后, 计算结果覆盖掉了一个操作数
- 一地址指令: 执行一次指只需访存两次, 取指令一次, 取操作数一次；被操作数和运算结果都放在累加寄存器中, 所以对于被操作数的读取和存入都不需要访存
- 零地址指令: 主要用在栈式计算机中, 参加运算的操作数从栈顶弹出, 运算后结果压入栈中
