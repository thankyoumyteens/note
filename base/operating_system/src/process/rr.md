# 轮转调度算法

轮转调度算法(Round Robin, RR), 也称为时间片轮转, 它的主要特点是将所有就绪状态的进程按到达时间排序, 然后为每个进程分配一个固定长度的时间片(quantum), 进程在其时间片内执行, 如果在时间片结束前没有完成, 它将被中断并放回就绪队列的末尾等待下一次调度。

RR 算法的核心是时间片, 它是进程可以连续执行的最长时间。时间片的长度可以从几毫秒到几百毫秒不等, 具体取决于系统的需求和设计, 对于时间片设置较短的情况, 可以快速响应用户交互和 I/O 事件。由于所有进程都按照一定的时间片轮流执行, RR 算法提供了很好的公平性, 确保每个进程都能获得 CPU 时间。

## 算法的优缺点

1. 优点
   - 提供了良好的响应性和交互性, 适合于需要快速响应的交互式系统
   - 保证了所有进程公平地获得 CPU 时间
   - 实现简单, 易于理解
2. 缺点
   - 时间片的选择对系统性能有很大影响, 需要根据实际情况进行调整
   - 可能会导致"颠簸(bouncing)"现象, 即进程频繁地在就绪队列和运行状态之间切换, 增加了上下文切换的开销
   - 对于长作业, 可能会因为频繁的中断而导致整体执行效率降低
