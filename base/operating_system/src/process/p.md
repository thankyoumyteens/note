# 优先级调度算法

优先级调度算法(Priority Scheduling)是操作系统中常用的一种 CPU 调度算法, 它根据进程的优先级来进行调度决策。在这种算法中, 每个进程都被赋予一个优先级值, 这个值通常是在进程创建时确定的。进程根据其优先级进行排序, 优先级高的进程将获得 CPU 的执行权。进程的优先级可以根据进程的行为或者系统策略进行动态调整, 例如, 一个等待 I/O 操作的进程可能会被降低优先级, 而一个长时间等待的进程可能会被提高优先级以避免饥饿。优先级调度可以是抢占式的, 也可以是非抢占式的。在抢占式优先级调度中, 如果一个更高优先级的进程变为就绪状态, 当前正在执行的进程将被中断, CPU 将被分配给新到达的高优先级进程。在非抢占式优先级调度中, 一旦进程开始执行, 它将一直运行直到完成或者自愿放弃 CPU。

## 算法的优缺点

1. 优点
   - 优先级调度算法可以为紧急任务提供快速响应, 特别是在抢占式模式下
   - 通过动态调整优先级, 系统可以适应不同的工作负载和性能需求
   - 优先级调度算法相对容易实现, 特别是在优先级可以通过数值大小直接比较的情况下
2. 缺点
   - 低优先级的进程可能会长时间得不到服务, 导致饥饿现象
   - 频繁的上下文切换和优先级调整可能会增加系统的开销
   - 为进程确定合适的优先级可能很困难, 特别是在没有足够信息的情况下
