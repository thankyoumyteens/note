# 互斥锁

- std::mutex（基本互斥锁）最基础的互斥锁，提供 lock()、unlock()和 try_lock()方法
  - 特点：同一线程只能锁定一次，再次锁定会导致死锁
  - 适用场景：保护简单的共享资源，防止多线程同时访问
- std::recursive_mutex（递归互斥锁）允许同一线程多次获取锁，需对应相同次数的解锁
  - 特点：解决同一线程递归调用时的死锁问题
  - 注意：过度使用可能掩盖设计问题，应谨慎使用
- std::timed_mutex（定时互斥锁）扩展了 try_lock_for()和 try_lock_until()方法，支持超时锁定
  - 特点：获取锁时可以指定等待时间，避免无限期阻塞
  - 适用场景：需要限制等待锁时间的场景
