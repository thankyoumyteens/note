# 第 5 课：结构化输出稳定性增强

第 4 课已经实现了：

```text
自然语言任务描述
  -> 模型输出 JSON
  -> Jackson 解析
  -> Java DTO
```

但这个版本还不够稳。模型可能返回 Markdown、解释文字、不合法 JSON、非法枚举值，或者字段为空。

本课目标是把流程升级为：

```text
模型输出
  -> 提取 JSON
  -> Jackson 解析
  -> 字段校验
  -> 失败后调用模型修复 JSON
  -> 再次解析
  -> 仍失败则抛出明确异常
```

一句话概括：

> 本课给结构化输出增加 JSON 提取、字段校验、自动修复和错误分类，让后端不再直接信任模型输出。

## 生产级结构化输出的基本链路

更稳的结构化输出链路应该是：

```text
Prompt 约束
  -> 模型输出
  -> JSON 提取
  -> DTO 解析
  -> 字段规范化
  -> 字段校验
  -> 失败自动修复
  -> 修复后再次解析
  -> 仍失败则返回明确错误
```

每一步都有作用：

| 步骤        | 作用                                   |
| ----------- | -------------------------------------- |
| Prompt 约束 | 尽量让模型输出正确 JSON                |
| JSON 提取   | 从 Markdown 或解释文字中提取 `{ ... }` |
| DTO 解析    | 把 JSON 转成 Java 对象                 |
| 字段规范化  | 空字符串转 null、默认值处理            |
| 字段校验    | 判断结果是否满足业务最低要求           |
| 自动修复    | 失败时让模型把错误输出修成合法 JSON    |
| 错误分类    | 让系统知道这是 AI 结构化输出失败       |
