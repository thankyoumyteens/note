# 第 3 课：实现流式输出接口

本课目标是让 AI Gateway 支持模型边生成边返回。

前两课中，普通聊天接口是：

```text
请求 -> 等待模型完整生成 -> 一次性返回完整答案
```

第 3 课升级为：

```text
请求 -> 模型生成一小段 -> 后端返回一小段 -> 前端实时展示
```

最终实现接口：

```http
POST /api/ai/chat/stream
```

## 为什么需要流式输出？

大模型生成长回答时，如果等完整生成后再返回，用户会感觉接口很慢。

例如模型生成一段 1000 字回答，可能需要几秒甚至十几秒。

普通接口的体验是：

```text
用户发送请求
等待...
等待...
等待...
一次性看到完整回答
```

流式接口的体验是：

```text
用户发送请求
马上看到第一段内容
继续看到第二段内容
继续看到第三段内容
直到生成完成
```

流式输出的价值：

- 用户体感更快
- 适合聊天产品
- 适合长文本生成
- 适合代码生成
- 适合 RAG 回答
- 适合后续 Agent 展示执行过程
- 适合前端做打字机效果
