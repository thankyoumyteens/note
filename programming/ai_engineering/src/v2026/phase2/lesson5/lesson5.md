# 第 5 课：结构化输出稳定性增强

## 本课目标

把现在的流程：

```text
模型输出 JSON -> Jackson 解析 -> 成功或报错
```

升级为：

```text
模型输出 JSON
  -> 清理 JSON
  -> Jackson 解析
  -> 字段校验
  -> 如果失败，自动让模型修复 JSON
  -> 再次解析
  -> 仍失败才返回明确错误
```

## 结构化输出的生产级基本链路

生产系统中的结构化输出不应该是：

```text
Prompt 写好一点 -> 直接相信模型
```

而应该是：

```text
Prompt 约束
  -> 模型输出
  -> JSON 清理
  -> DTO 解析
  -> 字段校验
  -> 默认值处理
  -> 自动修复
  -> 错误分类
  -> 日志记录
```

其中每一步都有作用。

| 步骤        | 作用                                |
| ----------- | ----------------------------------- |
| Prompt 约束 | 尽量让模型输出正确格式              |
| JSON 清理   | 处理 Markdown、解释文字、多余前后缀 |
| DTO 解析    | 把 JSON 转成 Java 对象              |
| 字段校验    | 判断对象是否满足业务要求            |
| 默认值处理  | 对缺失字段做合理兜底                |
| 自动修复    | 解析失败时让模型修复 JSON           |
| 错误分类    | 让调用方知道是 AI 输出问题          |
| 日志记录    | 便于排查模型异常输出                |
