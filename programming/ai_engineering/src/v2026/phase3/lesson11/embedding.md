# Embedding 是什么

**Embedding** 可以理解为：

```text
把一段文本转换成一串数字向量。
```

例如一句话：

```text
RAG 可以降低模型幻觉。
```

经过 embedding 模型后，会变成类似：

```text
[0.012, -0.083, 0.441, ..., 0.027]
```

这串数字不是给人看的，而是给机器用来计算“语义相似度”的。

---

## 为什么 RAG 需要 Embedding？

RAG 要解决的问题是：

```text
用户问了一个问题，系统怎么知道知识库里哪几段内容最相关？
```

如果只靠关键词匹配，效果很差。

比如用户问：

```text
怎么减少大模型胡编？
```

文档里写的是：

```text
RAG 可以降低模型幻觉。
```

这里“胡编”和“幻觉”不是同一个词，但语义接近。关键词搜索可能匹配不到，但 embedding 可以发现它们语义相关。

---

## Embedding 在 RAG 里的位置

你的第 11 课 RAG 流程是：文档上传 → 文本切分 → embedding → 向量入库 → top-k 检索 → LLM 基于 context 回答。

其中 embedding 出现两次：

```text
1. 文档入库时：
   chunk 文本 -> embedding 向量 -> 存入向量数据库

2. 用户提问时：
   question -> embedding 向量 -> 去数据库找最相似的 chunk
```

也就是：

```text
文档 chunk 的向量 和 用户问题的向量
        ↓
计算相似度
        ↓
找出最相关的 top-k 文档片段
```

---

## 一个直观类比

可以把 embedding 想成“语义坐标”。

比如：

```text
“苹果手机”
“iPhone”
“Apple 手机”
```

这些词表面不完全一样，但语义接近，所以它们的向量位置会比较近。

而：

```text
“苹果手机”
“红烧牛肉面”
```

语义差很远，向量距离就会比较远。
