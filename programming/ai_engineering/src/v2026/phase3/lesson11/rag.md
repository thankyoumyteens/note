# RAG 是什么

RAG 是 **Retrieval-Augmented Generation**，中文通常叫 **检索增强生成**。

简单说：

```text
先查资料，再让大模型回答。
```

普通大模型回答问题时，主要依赖它训练时学到的知识，容易有这些问题：

```text
不知道你的私有文档
不知道公司内部知识
知识可能过时
可能胡编
```

RAG 的做法是把回答流程改成：

```text
用户问题
  -> 先从知识库检索相关内容
  -> 把检索到的内容作为 Context 塞给大模型
  -> 大模型基于这些内容生成答案
```

比如你问：

```text
我们公司的报销规则是什么？
```

普通大模型可能不知道。而 RAG 系统会先去你的知识库里查：

```text
报销制度.pdf
财务流程.docx
员工手册.md
```

找到相关片段后，再让模型回答：

```text
根据公司报销制度，交通费需要在 30 天内提交发票和审批单……
```

它的核心价值是：

```text
让大模型能回答“外部知识 / 私有知识 / 最新知识”相关的问题。
```
