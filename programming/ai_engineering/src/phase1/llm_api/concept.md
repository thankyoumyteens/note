# 核心概念

- 角色 (Roles)：API 强制要求你把消息分类。
  - system：相当于给 AI 的“系统底层指令”或“人设”。它拥有最高优先级，决定了 AI 是个温柔的客服，还是个严谨的程序员，亦或是华尔街的毒舌交易员。
  - user：就是你（用户）发出的提问。
  - assistant：AI 之前的回答。为了让大模型有“记忆”，你需要把 user 和 assistant 的对话历史按顺序不断重新发给它（因为 HTTP 是无状态的）。
- Tokens：大模型不按“字”收费，按 Token 收费。一个 Token 大约相当于 0.7 个英文单词或半个汉字。请求的上下文越长，消耗的 Token 越多，API 调用的成本和延迟也越高。
- Temperature (温度)：控制输出的随机性。范围通常是 0 到 2。
  - 设为 0：极其严谨、确定。适合写代码、做数据信息提取。
  - 设为 0.7 - 1.0：有创造力、发散。适合写文章、或者扮演特定角色。
