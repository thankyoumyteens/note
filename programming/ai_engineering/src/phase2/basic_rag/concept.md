# 核心概念

## RAG 的六条流水线

这六个步骤是所有 RAG 系统的骨架，无论以后用到多复杂的框架（如 LlamaIndex 或 LangGraph），底层逻辑都不变：

1. 加载 (Load)：把各种格式的非结构化数据（PDF、Word、HTML、TXT）读取进内存。
2. 切分 (Split)：LLM 有上下文窗口限制（比如 8K 或 128K Token），而且整篇文章塞进去容易导致它“抓不住重点”。所以需要把长文切成一小块一小块（Chunk）。
3. 嵌入 (Embed)：这是 RAG 最核心的魔法。计算机不懂文字，Embedding 模型的作用就是把一段文本变成一个由浮点数组成的高维数组（向量）。
4. 存储 (Store)：把切好的文本块和对应的向量存进向量数据库（如 Chroma, FAISS, Milvus 等）。
5. 检索 (Retrieve)：当用户提问时，把用户的问题也变成向量，然后在数据库里计算距离（通常是余弦相似度），找出最接近的几个文本块。
6. 生成 (Generate)：把检索出来的文本块拼接到 Prompt 里，告诉大模型：“根据这段上下文，回答用户的问题。”

## 什么是 Embedding（嵌入）？

用 Java 的思维来理解：你可以把它当成语义的 HashCode。

传统的 `hashCode()` 只要字符串差一个字母，结果就天差地别。但 Embedding 是“空间坐标”：

- “苹果公司”和“iPhone”在多维空间中的坐标距离会非常近。
- “苹果公司”和“香蕉”的距离就会稍微远一点。
- 通过对比向量之间的数学距离，我们就能实现“懂意思”的搜索，而不仅仅是关键词匹配。
