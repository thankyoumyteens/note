# 可靠的模型输出

### 学习目标

从“模型能回答”推进到“输出可以被程序稳定消费”。

### 需要掌握

#### Prompt

- Prompt 模板设计
- 角色设定、任务约束和输出格式约束
- Few-shot 示例设计
- Prompt 版本管理
- Prompt A/B 测试与持续优化

#### 结构化输出

- 用户意图拆解与任务说明
- JSON、JSON Schema、枚举和字段约束
- Schema 校验、业务校验和错误反馈
- 输出不完整、格式错误和语义错误的处理
- 输出稳定性控制

#### 本阶段评估

- 建立小规模、可重复运行的测试集
- 记录结构化输出成功率、字段准确率和失败样本
- 对 Prompt 或模型变更执行回归测试

### 阶段产出与验收

- 将同一业务案例的自然语言请求转换为统一结构化对象
- 非法或缺失字段不会直接进入后续业务调用
- Prompt 和 Schema 变更有版本记录
- 能用固定测试集比较变更前后的结果，而不是只靠人工试问
