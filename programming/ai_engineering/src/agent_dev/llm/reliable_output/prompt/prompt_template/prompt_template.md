# Prompt 模板设计

Prompt 模板设计就是把一次 LLM 调用的提示词设计成一个稳定、可复用、可维护的“模板”，而不是每次临时手写一段自然语言。

你可以把它理解成：Prompt 模板 = 固定规则 + 变量输入 + 输出格式约束

它的目标不是“让模型聊得更好”，而是让模型在后端系统里表现得更稳定、更可控。

## Prompt 模板通常包含哪些部分

一个工程化 prompt 模板通常包含这些部分：

- 角色设定
- 任务说明
- 输入变量
- 上下文信息
- 约束规则
- 输出格式
- 示例
- 错误处理规则

例如：

```
角色设定：
你是一个订单系统的用户意图识别器。

任务说明：
根据用户输入，判断用户想执行的操作。

输入变量：
用户输入：{user_input}

约束规则：
- 只能返回 JSON
- 不要解释
- intent 必须来自指定枚举
- 如果无法判断，返回 UNKNOWN

输出格式：
{
  "intent": "QUERY_ORDER",
  "confidence": 0.95,
  "needClarification": false
}
```
