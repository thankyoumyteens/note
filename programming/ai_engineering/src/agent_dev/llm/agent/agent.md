# 最小可用 Agent

### 学习目标

让模型能够在受控边界内选择并调用工具，完成一个真实业务任务。

### 需要掌握

#### Tool Calling / Function Calling

- Tool Calling / Function Calling 原理
- Tool Schema 设计和工具描述边界
- 工具参数生成、Schema 校验和业务校验
- 外部 API 与企业内部系统集成，例如 Jira、飞书、KMS、PLM 和业务系统 API
- 工具调用结果解析、摘要和重新注入上下文
- 工具调用失败的分类、重试和终止
- 防止模型调用不存在、不适用或越权的工具

#### Agent 基础循环

- Agent 基本架构
- ReAct 模式和受限的“判断—调用—观察—继续”循环
- Tool、Context、State 和 Runtime 的基础职责
- Agent step 限制和终止条件
- Human-in-the-loop 人工确认机制

#### 本阶段治理

- 工具白名单、最小权限和高风险操作确认
- 工具输入、结果、耗时、异常和调用者记录
- 工具调用成功率、参数正确率和任务完成率评估

### 阶段产出与验收

- Agent 可以根据请求选择正确工具并完成至少一个端到端业务任务
- 所有工具参数在执行前经过校验
- 只读、可逆和高风险操作具有明确的权限与确认边界
- Agent 不会无限循环，失败后能返回可解释的终止结果
