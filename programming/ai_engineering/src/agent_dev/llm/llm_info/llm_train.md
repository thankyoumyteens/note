# LLM 是怎么训练出来的

LLM 通常经历几个阶段。

## 1. 预训练 Pre-training

用海量文本训练模型预测下一个 token

训练数据可能包括：

- 网页
- 书籍
- 论文
- 代码
- 文档
- 问答数据
- 论坛内容
- 技术博客
- 开源代码

预训练阶段学到的是：

- 语言规律
- 事实模式
- 代码模式
- 常识模式
- 推理模式
- 文本结构

例如模型在大量 Java 代码中看到了：

```java
@RestController
@RequestMapping
@GetMapping
@PostMapping
@Service
@Repository
```

它就会学到 Spring Boot 项目的常见写法。

但注意：

> 预训练学到的是统计模式，不是像数据库一样存储精确的事实。

## 2. 指令微调 Instruction Tuning

预训练模型只是会“续写文本”，但不一定听指令。

所以需要做指令微调，让它学会：

- 用户问问题 -> 正面回答
- 用户要求总结 -> 给总结
- 用户要求写代码 -> 写代码
- 用户要求解释 -> 解释

比如训练数据可能长这样：

```text
Instruction: 解释什么是 RAG
Answer: RAG 是检索增强生成……
```

经过这个阶段，模型就会从“文本续写器”变成“指令响应模型”。

## 3. 人类偏好对齐 RLHF / DPO

模型还需要学习什么回答更符合人类偏好。

比如同一个问题有两个回答：

- 回答 A：准确、清晰、有结构
- 回答 B：啰嗦、含糊、有错误

人类标注员或偏好模型会告诉模型：A 更好。

模型会继续调整，学会输出更符合人类期待的回答。

这就是为什么现代 ChatGPT / Claude 这类模型不仅会生成文本，还会尽量做到：

- 有帮助
- 有礼貌
- 较安全
- 结构清楚
- 避免明显危险内容

## 4. 工具调用训练 Tool Use Training

现在很多模型还会被训练成能输出工具调用意图。

例如用户问：今天洛杉矶天气怎么样？

模型自己其实不知道实时天气。但它可以判断：这个问题需要调用 weather 工具。然后输出类似：

```json
{
  "tool": "weather",
  "location": "Los Angeles"
}
```

注意这里很关键：

> 模型本身没有真的调用工具，它只是生成“我要调用哪个工具、参数是什么”。真正执行工具的是外部程序。

所以 Function Calling / Tool Calling 的本质是：

- LLM 负责决策和参数生成
- 程序负责真正执行
- 程序把结果返回给 LLM
- LLM 再整理成人类可读答案
