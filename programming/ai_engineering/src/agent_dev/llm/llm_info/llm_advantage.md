# LLM 擅长什么

## 1. 语言理解与生成

- 总结
- 改写
- 翻译
- 解释
- 分类
- 提取信息
- 生成文案
- 生成报告
- 问答
- 对话

比如：

- 把这段 JD 总结成技能清单
- 把这段中文改成自然英文
- 把错误日志解释成人话

这是 LLM 最强的区域。

## 2. 代码生成与代码解释

- 生成样例代码
- 解释代码逻辑
- 补单元测试
- 重构代码
- 定位常见 bug
- 生成接口文档
- 生成 SQL
- 生成脚本

但是代码生成不能盲信，必须配合：

- 编译
- 测试
- 静态检查
- 代码审查

**LLM 写代码像一个很强的初级/中级开发助手，但不是绝对可靠的资深工程师。**

## 3. 结构化信息抽取

LLM 很适合把非结构化文本转成结构化数据。

例如从招聘要求中抽取：

- 编程语言
- 框架
- 数据库
- Agent 技术
- 加分项

输出：

```json
{
  "languages": ["Java", "Python"],
  "frameworks": ["LangChain", "LangGraph", "Spring Boot"],
  "databases": ["PostgreSQL", "Milvus", "Elasticsearch"]
}
```

但工程上必须做：

- JSON Schema 约束
- 字段校验
- 异常处理
- 重试机制

## 4. 模糊任务拆解

LLM 很擅长把模糊需求拆成步骤。

比如：我要做一个企业知识库问答系统。

它可以拆成：

1. 文档上传
2. 文档解析
3. 文档切片
4. Embedding
5. 向量入库
6. 用户提问
7. 检索相关 chunk
8. 拼接上下文
9. 调用 LLM
10. 返回答案

这就是 Agent 里 Planner 的基础。

## 5. 语义匹配

LLM 和 Embedding 模型擅长处理语义相似。

例如：

- 用户问：如何提高召回效果？
- 文档写：优化检索命中率的方法

传统关键词搜索可能不一定匹配。

语义检索则可以判断：“召回效果”和“检索命中率”相关。

这就是 RAG 的基础能力。

## 6. 多领域迁移

LLM 可以快速适配不同场景：

- 客服
- 金融
- 教育
- 编程
- 法律辅助
- 医疗辅助
- 办公自动化
- 数据分析
- 研发提效

但注意：

> 可以快速适配，不代表天然专业可靠。垂直领域必须加知识库、规则、评估和人工审核。
