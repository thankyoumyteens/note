# 断流处理

断流分两种：**客户端断流** 和 **服务端断流**。

## 客户端断流

例如：

- 用户关闭页面
- 用户刷新页面
- 浏览器网络断开
- 前端主动 abort

这时后端应该停止继续生成，释放资源。

否则会出现：

- 前端已经不要了
- 后端还在调用模型
- provider 还在持续计费

最佳实践：

- 检测客户端取消
- 取消上游 WebClient 请求
- 停止模型流
- 记录 cancelled 状态
- 不要当成系统错误告警

客户端取消不是错误，它是正常业务行为。

## 服务端断流

例如：

- LLM Provider 连接断开
- provider 返回异常事件
- 后端处理异常
- Nginx 超时断开
- 服务重启
- 网络抖动

这时要区分：

1. 已输出部分内容
2. 还没输出任何内容

如果还没开始输出，可以走普通 HTTP 错误响应。

但如果 SSE 已经开始输出了，HTTP status 已经是 200，后面就不能再改成 500 了。

这时只能通过 SSE 事件告诉前端：

```text
event: error
data: ...
```

然后再发送结束事件或直接关闭连接。
