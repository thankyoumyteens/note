# 什么是 Streaming

普通非流式调用是这样：

1. 用户发问题
2. 后端请求模型 API
3. 模型生成完整答案
4. 一次性返回完整 JSON
5. 前端显示完整答案

缺点：

1. 首屏等待时间长
2. 用户不知道模型是不是卡住了
3. 长答案体验很差

Streaming 是这样：

1. 用户发问题
2. 后端请求模型 API，stream=true
3. 模型生成一个 token / chunk 就返回一个 chunk
4. 后端持续接收 chunk
5. 后端持续推给前端
6. 前端边收边显示

视觉效果就是 ChatGPT 那种：

```text
这
这 是 一
这 是 一个 流 式
...
```
