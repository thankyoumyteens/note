# 基本概念介绍

## 大语言模型 (LLM)

你可以把它当成一个读完了全网人类知识（截止到它训练完成那一天）的超强实习生。

- 它的优点：语言组织能力极强，逻辑推理能力强，懂很多通用知识（比如知道什么是 Spring Boot，什么是市盈率）。
- 它的致命弱点：
  - 没有记忆（无状态）：你每次调用它，它都像第一次认识你。所以我们刚才要用 RunnableWithMessageHistory 把聊天记录当成参数传给它。
  - 没有私有数据：它绝对不知道你们公司内部的 API 文档，也不知道你们“508班”的新年晚会有什么节目。
  - 会“幻觉”（胡说八道）：如果你问它不知道的事，它为了讨好你，会一本正经地瞎编。

## 词嵌入 (Embedding)

计算机不懂汉字，只懂数字。Embedding 就是把一段文字变成一组高维坐标（向量）。最神奇的是，它包含了“语义（含义）”！

比如在向量空间里，“股息率”和“分红回报”这两个词在字面上没有任何重合，但它们的 Embedding 数组通过数学计算（余弦相似度）得出的距离会非常非常近。而“股息率”和“苹果公司”的距离就会很远。

传统 Java 里判断两个字符串是否一样用 equals()，或者用 hashCode() 算出一个整数型哈希值。而 Embedding 是一个高级的 hashCode() 函数，它返回的不是 int，而是一个浮点数数组（`float[]`），比如长度为 1536 的数组。

## 向量数据库 (Vector Database)

传统的 MySQL 是精确查找（`WHERE name = '张三'`），ES 是倒排索引（关键词命中）。

向量数据库里面存的全是上面提到的 `float[]` 数组。当你问“Palantir 的估值方法是什么？”，系统会把你的问题也变成一个数组，然后去数据库里找 **“距离最接近”** 的几个数组，把它们对应的原文拿出来。这就是为什么你不用精确的关键词，它也能帮你把那段关于 P/S 和自由现金流的研报找出来。

## RAG (检索增强生成)

刚才你写的整个 main.py 就是一个 RAG 系统。它的工作流极其像传统后端的查库组装逻辑：

1. 前端请求：用户问“晚会有几个节目？”
2. 检索库 (Retrieve)：系统拿着问题去向量数据库里查，找出最相关的段落（“共有24个节目...”）。
3. 模板增强 (Augment)：系统把你写好的 Prompt 模板拿出来，把查到的段落填进 `<context>` 标签里。
4. 生成结果 (Generate)：把这整段拼接好的长文本发给 LLM（超级实习生），跟他说：“别瞎编，只准看着这段材料回答。”LLM 乖乖提炼出结果，返回给用户。

## Agent (智能体)

RAG 只是让 AI 去“读”静态数据。而 Agent 是让 AI 有手有脚去“做”事。

假如你给了 LLM 几个工具（其实就是用 Python 写几个函数，比如 `getRealTimeStockPrice(String ticker)` 和 `calculateBogle(float pe, float roe)`）。

当你问 Agent：“帮我算一下 TSLA 今天的博格公式预期回报”，Agent 的大脑（LLM）会自己思考：

1. “第一步，我得先调用 `getRealTimeStockPrice("TSLA")` 拿到实时数据。”
2. “第二步，我把拿到的数据传给 `calculateBogle()`。”
3. “第三步，我把计算结果整合成一段话回复给用户。”

这就是 Agent：大模型不仅生成文本，还负责做决策和路由（Routing）。
