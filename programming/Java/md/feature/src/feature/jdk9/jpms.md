# 模块化系统

Java 9 引入的模块化系统（JPMS，Java Platform Module System，代号 Jigsaw） 是 Java 平台最重要的变革之一，旨在解决传统类路径（Classpath）的缺陷，为大型应用提供更清晰的依赖管理、更强的封装性和更高效的资源隔离。

传统类路径（Classpath）存在三大痛点：

1. 依赖混乱：所有类文件平级存放，无法明确声明 “应用依赖哪些库”，容易出现 “类缺失” 或 “版本冲突”（如 NoClassDefFoundError）
2. 封装失效：public 修饰的类在全局可见，即使是内部实现类也可能被外部错误依赖，导致代码脆弱性（修改内部实现可能影响外部）
3. 资源冗余：JRE 包含大量不常用的类和资源（如所有本地化数据、未使用的模块），无法按需裁剪，导致运行时体积庞大

JPMS 通过 “模块” 的概念从根本上解决这些问题。
