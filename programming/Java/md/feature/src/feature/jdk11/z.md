# ZGC

Java 11 引入了 ZGC（Z Garbage Collector）, 一款实验性的低延迟垃圾回收器，专为大堆内存（TB 级） 和低延迟需求设计，目标是将垃圾回收的停顿时间控制在 10 毫秒以内，同时支持大规模内存场景，填补了 Java 在超大型应用和低延迟领域的空白。

## ZGC 的核心设计目标

- 超低延迟：GC 停顿时间不超过 10 毫秒（实际测试中常低于 1 毫秒），几乎不影响应用的响应性
- 支持大堆内存：轻松应对从几百 MB 到数 TB 的堆内存，满足大型应用（如数据库、大数据处理）的需求
- 高吞吐量：在保证低延迟的同时，尽量减少对应用吞吐量的影响（吞吐量略低于 G1，但远高于 CMS）
- 跨平台支持：最初支持 Linux/x86-64，后续扩展到 Windows、macOS 等平台
