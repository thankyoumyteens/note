# 引用类型

有三种 `reference` 类型: class 类型, array 类型, 和 interface 类型. 它的值引用的是动态创建的类实例, 数组, 或者实现了某些接口的类实例或数组。

一个数组类型由一个单一维度的 _组件类型(component type)_ 组成 (whose length is not given by the type). 一个数组类型的组件类型本身可能也是数组类型。如果，从任何数组类型开始, 里层是组件类型, 然后(如果还是数组类型)就是更里层的组件类型, 以此类推, 最终, 一定会找到一个不是数组的组件类型; 它被称为数组的 _元素类型(element type)_ 。数组的元素类型一定是以下类型之一: 原始类型, 类, 或者接口。(比如三维数组 `int[][][]` 的组件类型为 `int[][]`, 二维数组 `int[][]` 的组件类型为 `int[]`, 一维数组 `int[]` 的组件类型是 `int`。)

一个 `reference` 类型的值也可以是特殊值 null 的引用, 不指向任何对象的引用, 表示为 `null`。`null` 引用一开始没有运行时类型, 但是可以转换成任意类型。`reference` 类型的默认值就是 `null`。

本规范不强制要求编码 `null` 的具体值。
