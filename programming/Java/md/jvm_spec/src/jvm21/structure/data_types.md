# 数据类型

和 Java 类似, JVM 操作的数据类型分为两种: 原始类型(primitive types)和引用类型(reference types)。

数据类型的检查需要在 JVM 运行之前完成, 一般是编译器去做这项工作。在运行时, 原始类型的值不需要额外的标记或检查, 以区别它和引用类型的值。因为 JVM 字节码指令的操作数会指明它需要操作什么类型的数据。例如, _`iadd`_, _`ladd`_, _`fadd`_, 和 _`dadd`_ 都是对两个数做加法的指令, 但是它们分别需要不同的操作数类型: `int`, `long`, `float`, `double`。

JVM 显式支持对象。一个对象可以是动态分配的类实例, 也可以是数组。一个指向某个对象的引用, 在 JVM 中称为引用类型。引用类型的值可以理解为指向对象的指针。一个对象可能会有多个引用, 对象总是通过引用类型进行操作、传递和验证。
