# 入门

## 安装Cursor

[https://cursor.com/](https://cursor.com/)

## 操作流程

安装好 Cursor 后，别急着写代码。记住 Vibe Coding 的核心循环：提需求 -> 看结果 -> 甩锅 (报错就扔给 AI)。

### 1. 创建项目

打开 Cursor，新建一个空文件夹。按下 Cmd+I (Mac) 或 Ctrl+I (Windows) 打开 Composer 窗口。

### 2. 用自然语言写“需求文档”

不要想代码怎么写，要把自己当成产品经理。

- ❌ 错误示范：写一个 HTML 文件，里面有一个 div，id 是 app...（太细了，这是程序员思维）
- ✅ 正确示范：帮我做一个网页版的贪吃蛇游戏。背景是黑色的，蛇是绿色的。要在屏幕右上角显示分数。用 HTML/CSS/JS 实现，保存为单文件。

### 3. 运行与验收 (Vibe Check)

AI 写完后，你直接打开生成的 HTML 文件查看效果。

### 4. 遇到 Bug 怎么办？(核心心法)

这是 Vibe Coding 和传统编程最大的区别。千万不要尝试自己去读代码找 Bug。

1. 复制报错信息：如果在终端看到红色的字，或者网页里弹窗报错，直接 Ctrl+C 复制。
2. 粘贴给 AI：打开 Cursor 对话框，粘贴报错信息，然后打两个字：修好 (Fix it)。
3. 如果没报错但没反应：直接用大白话描述现象：
   - “蛇不动了。”
   - “点击按钮没反应。”

记住：在 Vibe Coding 里，报错信息不是给你看的，是给 AI 看的。你只是一个搬运工。

### 5. 快速迭代 (像捏泥巴一样改软件)

一旦程序跑通了，就开始发挥你的想象力，通过不断的“对话”来完善它。这就是 Vibe Coding 最爽的阶段。

你可以尝试这样发指令：

- 改样式：把界面做得更有赛博朋克感，用霓虹配色。
- 加功能：加一个“无敌模式”，蛇撞墙不会死。
- 改适配：我想在手机上也能玩，帮我适配一下触屏操作。

## 避坑建议

1. 步子不要迈太大：
   - 不要上来就说 给我做一个淘宝。AI 会懵。
   - 要拆解：先做一个登录页 -> 再做一个商品列表 -> 再做购物车。
2. 勤提交 (Save/Commit)：
   - 如果现在的版本跑得很好，立刻把文件备份一下（或者用 Git 提交）。
   - 因为有时候 AI 改着改着，可能会把原本好的功能改坏了（Vibe 坏了）。
3. 保持英语/中文混用：
   - 虽然国产大模型中文很好，但像 Claude/GPT-4o 这种模型，用英语描述技术名词通常更精准。
   - 比如：用 Tailwind CSS 写样式 比 用那个原子化的 CSS 框架 效果更好。
