# Vibe Coding

**Vibe Coding** 是最近在科技圈（特别是 X/Twitter 和 AI 开发者社区）非常火的一个概念，最早主要由前特斯拉 AI 总监、OpenAI 创始成员 **Andrej Karpathy** 带火。

简单来说，**Vibe Coding 指的是一种“随性、靠感觉、完全依赖 AI”的编程方式。**

在这种模式下，人类不再关注具体的代码语法、分号有没有写对、库有没有引对，而是专注于**自然语言的描述**和**最终的运行效果**。

## 核心定义

- **人类的角色**：产品经理 + 验收员。你只需要用自然语言（英语或中文）告诉 AI 你想要什么功能，或者你希望软件长什么样。
- **AI 的角色**：资深程序员。AI（如 Claude 3.5 Sonnet, GPT-4o）负责处理所有的逻辑实现、代码编写、错误调试和环境配置。
- **“Vibe”的意思**：这里的 Vibe（氛围/感觉）指的是你只需要关注程序运行起来“感觉对不对”。如果跑起来报错，直接把错误扔给 AI 让它修；如果界面丑，告诉 AI “让它好看点”。你是在和 AI 进行一种基于“感觉”的协作，而不是基于严谨的逻辑推导。

## 这个词的来源

Andrej Karpathy 在社交媒体上分享了他的编程体验，表示他现在大部分时间都在写英语（Prompt），而不是写 Python 或 C++。他说：

> "I just see stuff, say stuff, run stuff, and copy paste errors."
> (我只是看着屏幕，说出想法，运行代码，然后复制粘贴错误信息。)

这种编程方式极其流畅，不需要停下来查文档，就像在“Vibing”（享受氛围/随波逐流）一样，所以被称为 Vibe Coding。

## 总结

**Vibe Coding = 只有想法的你 + 负责干活的 AI。**

它标志着编程门槛的再一次大幅降低，让不懂代码的人也能体验“构建者”的快乐。你不需要懂 `for loop`，你只需要懂你想要做什么。
