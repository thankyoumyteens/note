# zip 文件的结构

ZIP 文件是存储多个文件的存档(JAR 本质上就是一种带额外约定的 ZIP)。ZIP 允许使用许多不同的方法压缩包含的文件，以及简单地存储文件而不压缩它。每个文件都单独存储，允许使用不同的方法压缩同一存档中的不同文件。由于 ZIP 存档中的文件是单独压缩的，因此可以提取它们或添加新文件，而无需对整个存档应用压缩或解压缩。

zip 文件包含两部分:

1. 许多 entry(每个文件/文件夹对应一个 entry)
2. 中央目录

```
Enrty1
Entry2
Entry3
...
中央目录
```

中央目录记录了 ZIP 文件中的各个文件(entry)的位置以及大小等信息，位于 ZIP 文件的末尾。中央目录允许 ZIP 读取器加载文件列表，而无需读取整个 ZIP 存档。将中央目录存储在 ZIP 的末尾还可以实现通过将压缩文件附加到无害的文件（如图片文件）来隐藏压缩文件。

ZIP 文件的开头还可以包含与 ZIP 存档无关的额外数据。比如将将文件标记为可执行文件，从而将 ZIP 制作成自解压存档。
