# ZipKin

Zipkin 有 4 个核心组件:

- Collector：收集器组件, 它主要用于处理从外部系统发送过来的跟踪信息, 将这些信息转换为 Zipkin 内部处理的 Span 格式, 以支持后续的存储、分析、展示等功能
- Storage：存储组件, 它主要对处理收集器接收到的跟踪信息, 默认会将这些信息存储在内存中, 可以修改此存储策略, 通过使用其他存储组件将跟踪信息存储到数据库中
- RESTful API：API 组件, 它主要用来提供外部访问接口。比如给客户端展示跟踪信息, 或是外接系统访问以实现监控等
- UI：基于 API 组件实现的上层应用。通过 UI 组件用户可以方便而有直观地查询和分析跟踪信息

zipkin 分为服务端和客户端, 服务端主要用来收集跟踪数据并且展示, 客户端主要功能是发送给服务端, 微服务的应用也就是客户端, 这样一旦发生调用, 就会触发监听器将 sleuth 日志数据传输给服务端。
