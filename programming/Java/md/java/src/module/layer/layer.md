# 层

层(Layer)对于模块, 就像类加载器对于类那样: 一种加载和实例化的机制。

Layer 的核心目的是在运行时对模块进行分组管理，解决了传统模块化中 “一次性加载所有模块” 的局限性，支持更灵活的模块生命周期控制。它的作用包括:

- 动态模块加载：允许在应用运行时动态加载新的模块（无需重启应用）
- 模块隔离：不同 Layer 中的模块可以相互隔离（例如类加载隔离），避免冲突
- 依赖管理：每个 Layer 可以有自己的模块依赖关系，且可以基于已有的 Layer 扩展（形成父子关系）

典型使用场景:

- 插件系统：应用核心模块在基础 Layer 中，插件模块动态加载到新的 Layer 中，实现插件的热插拔
- 分阶段加载：大型应用按功能拆分模块，启动时先加载核心模块（基础 Layer），后续按需加载扩展模块（新 Layer）
- 模块版本隔离：不同 Layer 可以加载同一模块的不同版本，满足兼容性需求
