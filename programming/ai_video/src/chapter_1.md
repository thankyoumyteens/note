# 学习路线

## 阶段一：基建与构建“唯一真理源” (Source of Truth)

**目标：放弃直接画视频，先产出绝对完美的静态参考图。**

在 2026 年的 I2V（图生视频）工作流中，第一张图决定了整个视频的基因。我们需要把它当作数据库里的“主键”来构建。

- **核心引擎：ComfyUI 的现代化部署**。不要把它当成一个画图软件，把它看作一个基于 Python 的有向无环图（DAG）任务编排引擎。这套原生 Python 肌肉记忆对你未来的全栈开发极其重要。
- **静态底模与微调 (FLUX / SDXL + LoRA)**：使用目前对语义理解最精准的模型（如 FLUX 或 SDXL 最新版）。这一步的唯一目的，是生成一张你最满意的卡提希娅“定妆照”。
- **小里程碑**：在 ComfyUI 中跑通基础流，生成一张没有任何结构崩坏、材质光影完美的卡提希娅静态图。这张图将作为后续所有视频生成的“绝对真理源”。

## 阶段二：动态特征捕获 (Motion Extraction)

**目标：从真实物理世界提取动作骨架，剥离冗余数据。**

不要再用旧版 ControlNet 去锁单张图的边缘了。我们需要的是连续的运动向量。

- **视频预处理管道 (Video Preprocessing)**：学习使用 `ComfyUI-VideoHelperSuite` 导入真实人物走路的视频。
- **高精度姿态估计 (DWPose for Video)**：利用你的显卡算力，逐帧精准提取真人视频中的火柴人骨架序列。这一步就像是数据清洗，过滤掉背景和真人外貌，只保留纯粹的运动轨迹。
- **小里程碑**：成功将一段 5 秒的真人走路视频，转化为极其丝滑、没有一帧断裂的黑色底片彩色骨架视频。

## 阶段三：原生 DiT 视频生成 (The Core Engine)

**目标：结合静态真理源与动态骨架，进行原生时序渲染。**

这是与旧版 AnimateDiff 产生代差的核心阶段。我们直接调用为视频而生的原生大模型。

- **本地核武 (Wan2.7 / HunyuanVideo)**：加载 2026 年最新的原生 DiT 开源视频大模型。它们天生具备三维空间一致性和物理规律理解。
- **动作克隆与特征映射 (MimicMotion 架构)**：在这个工作流中，你不再需要写冗长的提示词来描述人物。模型会直接读取阶段一的“静态图”作为材质包，读取阶段二的“骨架视频”作为驱动引擎，自动渲染出具有完美时序一致性的动态画面。
- **小里程碑**：生成一段 5 秒的卡提希娅走路视频。衣服纹理、头发长度在 64 帧内完美锁死，背景不再出现红蓝闪烁，动作严格吻合真人骨架。

## 阶段四：音频对轨与业务层封装 (API Integration)

**目标：声画同步，并将其沉淀为可被外部调用的微服务。**

视频生成不仅是视觉的艺术，更是工程的落地。

- **现代唇形同步 (EchoMimic / 极速版 Wav2Lip)**：输入角色配音音频，驱动生成的视频进行口型重绘。2026 年的工具已经可以做到连面部微表情（如眨眼、挑眉）都与音频情绪完美对齐。
- **系统级封装**：利用 ComfyUI 的 API 模式，将你搭建好的“静态图 + 动作视频 + 音频 = 最终视频”工作流封装成一个 JSON 接口。这可以作为你未来 AI 业务系统的一个核心底层能力。
- **小里程碑**：不仅完成一部带配音的二次元短片，还能通过纯 Python 脚本向本地 ComfyUI 服务发送请求，实现视频生成的自动化批处理。
