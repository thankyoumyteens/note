# 克隆声音

简单来说，只需在“Input Text”输入目标台词，在“Reference Audio”选项卡上传一段纯净的原声录音并填入其原话，最后点击“Generate”，即可利用本地算力瞬间克隆出高度还原的语音。

### 🎙️ 核心流程：零样本语音克隆

要在本地实现高质量的音色复刻（例如还原特定游戏角色的声线），请按照以下步骤操作：

**1. 输入目标文案 (Input Text)**
在界面左上角最大的文本框内，输入你希望模型最终读出来的全新台词。

**2. 配置参考素材 (Reference Audio Tab)**
点击界面中部的 `Reference Audio` 标签页，这里是控制音色和语气的核心：

- **上传录音 (Reference Audio)**：将 5 到 10 秒的参考音频（`.wav` 格式最佳）拖拽到上传框内。**关键点**：音频必须尽可能“干”，若带有环境音或 BGM，模型会将其误认为声带特征一并学习，导致生成的音频带有电子杂音。
- **参考文本 (Reference Text)**：**这一步极其重要**。必须一字不差地填入上传音频中角色所说的原话（包括标点符号的停顿）。若音频内容与填入的文本不匹配，模型的交叉注意力机制会发生错乱，导致直接输出空白或崩溃。
- **Memory Cache (内存缓存)**：保持默认的 `on` 即可。这会将提取出的声纹特征缓存在显存中，当你连续生成同一角色的多句台词时，速度会大幅提升。

**3. 生成与下载 (Generate)**
点击右侧显眼的蓝色 `Generate` 按钮。随后可以在 `Generated Audio` 框内直接试听，或点击右侧小图标下载到本地。

### ⚙️ 进阶调优：大模型参数解析 (Advanced Config)

当你发现生成的语音语气不够生动，或者出现了大模型常见的“幻觉”时，请点击切换到 `Advanced Config` 标签页进行微调。这里的底层逻辑与文本 LLM 高度一致：

- **Top-P (默认 0.8)**
  - **作用**：核采样阈值，控制发音库的概率分布。
  - **调整建议**：如果你觉得声音偏机械、缺乏感情，可以适当调高（如 0.85-0.9）增加语气丰富度；如果模型开始出现“乱颤”或“吃字”，说明分布过广，需要调低。
- **Temperature (默认 0.8)**
  - **作用**：控制生成的随机性和创造力。
  - **调整建议**：通常与 Top-P 配合调整。调高能让角色的语调变化更生动，调低则让输出更稳定、符合参考音频的基准频率。
- **Repetition Penalty (默认 1.1)**
  - **作用**：重复惩罚。
  - **调整建议**：如果生成的语音出现“结巴”，或者在句末疯狂重复读某几个字（大模型常见病），请将此值适当调高至 1.15 或 1.2 来强制模型跳出循环。
- **Iterative Prompt Length (默认 300)**
  - **作用**：生成长文本时的切分上下文长度。
  - **调整建议**：填 0 则关闭迭代。如果你一次性输入了几百字的长段落，保持默认值可以让模型分段处理，避免显存溢出或后半段音色崩坏。
- **Maximum tokens per batch (默认 0)**
  - **作用**：批处理上限。
  - **调整建议**：默认 0 表示无限制，利用 32GB 的大显存可以直接拉满处理，无需修改。
- **Seed (默认 0)**
  - **作用**：随机种子。
  - **调整建议**：0 表示每次生成都会采用不同的随机数，哪怕参数相同，语气也会有细微差别。如果你刷出了一句语气极其完美的音频，可以通过控制台查看其生成的具体 Seed 值并填入这里，以便后续锁定这种完美的语气状态。
