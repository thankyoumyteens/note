# 阶段二：动态特征捕获

如果说阶段一是“建立材质数据库主键”，那么阶段二就是典型的 **“数据清洗与信号提取 (Data Cleaning & Signal Extraction)”**。

找一段真人正面对着镜头走路的视频，这段视频包含着大量我们 **不需要的“脏数据”**（真实的背景、真人的衣服、真人的长相）。如果直接把这些数据喂给视频大模型，它会产生严重的认知混乱（“我到底是该画卡提希娅，还是画这个真人？”）。

我们要用特定的算法层，把这层“物理外壳”彻底剥离，只提取出纯粹的 “运动坐标矩阵”（也就是纯黑背景下的彩色火柴人骨架）。

_(为了保证模块化开发，请在 ComfyUI 中 **新建一个空白画布**)_。

## 1. 代理与网络环境准备 (仅限本地 Windows)

由于我们需要下载数十 GB 的各种控制模型，如果在本地部署且使用了网络代理软件，必须让 Python 环境也走代理，否则会出现严重的下载中断或 `Connection Error`。

打开你的 Anaconda Prompt，激活环境并配置网络：

```cmd
call conda activate ai_demo

# 安装支持 Socks5/HTTP 代理的扩展包
pip install httpx[socks]

# 安装完成后重新启动 ComfyUI
```

## 2. 安装核心插件

在 ComfyUI 网页界面（`http://127.0.0.1:8188`）的右侧控制面板，点击 **Manager** 按钮，然后点击 **Install Custom Nodes**。

在搜索框中搜索并安装以下两个核心组件：

1.  **`comfyui_controlnet_aux`** (作者 Fannovel16)：最强大的视觉信号提取库。
2.  **`ComfyUI-VideoHelperSuite`** (作者 Kosinkadink)：处理视频流的分帧与合成神器。

_安装完成后，记得点击 `Restart` 重启 ComfyUI 以加载插件。_

## 3. 构建 DAG 拓扑 (节点连线)

在空白区域双击左键即可打开搜索框，输入关键字搜索对应的模块。

1.  **`Load Video (Upload) 🎥🅥🅗🅢`**
    - **操作**：点击 `choose video to upload`，把你那段“人物背对着走路”的 MP4 视频传上去。
    - **参数配置**：找到 `frame_load_cap`（最大加载帧数），把它设为 **`64`**（这是目前开源 DiT 视频模型的黄金测试帧数，大约对应 2~3 秒的流畅运动，既能验证效果又不会让 5090 等太久）。
2.  **`DWPose Estimator`**
    - _（注：这是 2026 年最顶级的姿态估计模型，远超老旧的 OpenPose，它能极其精准地捕获手指微小关节和面部朝向）。_
    - **连线**：将 `VHS_LoadVideo` 的 **`IMAGE`** 输出 -\> 连入此节点的 **`image`** 输入。
    - **参数配置**：保持默认即可（它会自动加载 `yolox` 框选人物，用 `dw-ll` 计算骨架）。
3.  **`Video Combine 🎥🅥🅗🅢`**
    - **连线**：将 `DWPose Estimator` 的 **`IMAGE`** 输出 -\> 连入此节点的 **`images`** 输入。
    - **参数配置**：把 `frame_rate`（帧率）设为和你原视频一样的数值（通常是 `24` 或 `30`）。`format` 选 `video/h264-mp4` 以便直接在网页里预览。

## 4. 运行与处理“假报错”

点击右上角的 **`Run`** 开始运行。

### ⚠️ 首次运行的必然报错处理

如果是第一次运行，你大概率会看到一个红色的报错弹窗：

> `ERROR: model dw-ll_ucoco_384_bs5.torchscript.pt not found...`

**原因分析：**
这是因为 Manager 插件在请求发送前做了一次 **“前置静态校验 (Pre-flight Check)”**。它扫描了你的硬盘，发现找不到对应的模型文件，于是直接拦截了队列。但 Manager 不知道的是，`DWPose Estimator` 这个节点内部写了 **“运行时动态拉取 (Runtime Lazy-loading)”** 的逻辑。只要请求打到它那里，它就会自己去 HuggingFace 下载。

**解决方案：**
直接点击报错弹窗右下角的 **`Queue Anyway` (无论如何都要排队)** 按钮强行通过校验！

_(注意：在本地 Windows 环境下，此时可以切回你的 Anaconda Prompt 黑框窗口，你会看到进度条正在下载模型。请耐心等待，如果下载报错，请检查第 1 步的代理设置是否生效。)_

## 5. QA 测试 (Quality Assurance)

模型下载完毕后，你的 RTX 5090 将动用 Tensor Core 开始逐帧扫描视频。

**小里程碑验收标准：**
跑完之后，在最右侧的 `Video Combine` 节点里，你会看到一个在纯黑背景下走路的彩色火柴人视频。
请用 QA 的眼光仔细检查这段提取出来的信号：

1.  **有没有断帧/闪烁？**（比如走到一半突然火柴人消失了，说明原视频该处的光线或遮挡太严重，算法丢失了目标。必须换原视频）。
2.  **手脚是否完整？**（如果原视频里脚被裁掉了，骨架也会没有脚，这会导致后续卡提希娅生成的腿部直接畸形崩溃）。
3.  **手指清晰吗？** DWPose 应该能捕捉到手指的细节，如果火柴人手部只有一团，说明原视频太糊了。

当你得到一段极其丝滑、完整的“骨架动作捕获”视频时，恭喜你，最枯燥的数据清洗阶段结束了。保存这段纯净的骨架，准备进入最激动人心的原生视频生成环节！
