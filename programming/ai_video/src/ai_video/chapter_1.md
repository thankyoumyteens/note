# 本地 AI 视频生成学习路线

**前言**
本路线图针对本地拥有充足算力的开发者。核心思路是：前端使用 ComfyUI 固定画面构图，后端通过 Python 脚本直接调用大模型生成视频，避免在 ComfyUI 中处理复杂的视频逻辑。

---

## 阶段一：生成静态参考图

**目标：生成一张比例准确、细节完整的静态图，作为视频的起始帧。**

视频的质量高度依赖第一张图的质量。我们需要确保起始帧没有任何结构性错误。

- **明确工具定位：** 仅将 ComfyUI 作为静态图像生成工具，不在此处理时序和动态效果。
- **图像质量控制：** 使用 FLUX 或 SDXL 模型。配合目标检测（YOLO）和面部修复（FaceDetailer）节点，确保人物五官清晰、肢体和物理比例正确。
- **阶段产出：** 一张光影正确、没有结构崩坏的高分辨率人物静态图。

---

## 阶段二：配置本地运行环境

**目标：配置专用于运行百亿参数视频大模型的 Python 开发环境。**

- **环境隔离：** 使用 Conda 创建独立的虚拟环境，并安装与硬件相匹配的 CUDA 和 PyTorch 版本。
- **核心依赖：** 安装并熟悉 HuggingFace 的 `diffusers` 库，作为调用大模型的基础工具。
- **本地模型：** 将所需的视频大模型（如 Wan 2.2 等）权重文件完整下载到本地固态硬盘中。

---

## 阶段三：使用代码生成视频

**目标：编写 Python 脚本，直接读取静态图并调用视频模型生成视频。**

直接调用大模型可以更好地利用其自带的空间理解和运动生成能力。

- **编写推理脚本：** 使用 `diffusers` 库编写极简的 Python 脚本。加载第一阶段生成的静态图路径，并输入动作提示词（例如 `walking away slowly`）。
- **模型生成：** 依靠模型自身的参数能力处理物理运动、衣物摆动和光影变化，避免画面形变或闪烁。
- **阶段产出：** 在终端运行脚本，输出一段动作流畅的 5 秒动态视频。

---

## 阶段四：音频驱动与口型同步

**目标：为生成的视频添加配音，并使人物口型与音频匹配。**

- **引入音频模型：** 在代码流程中集成音频驱动模型。
- **面部重绘：** 输入配音音频文件，利用模型根据音频重绘视频中的面部区域，实现口型同步和微表情变化。
- **阶段产出：** 将视频与音频合并，导出一支口型匹配的完整有声短片。
