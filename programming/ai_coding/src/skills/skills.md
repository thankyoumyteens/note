# SKILL

**Skill（技能）** 是指 Agent 为了完成特定任务而具备的**执行能力**或**工具集**。

如果把大模型（LLM）比作 Agent 的“大脑”，那么 Skill 就是它的“手脚”或“专业工具箱”。

## Skill 的本质：能力封装

在技术层面，一个 Skill 通常是一个被封装好的**函数（Function）**、**API 接口**或**一段特定代码**。Agent 通过“规划”意识到自己无法仅靠大脑知识解决问题时，就会调用这些 Skill。

## Skill 的主要分类

- **工具调用类 (Tool Use)：**
  这是最常见的 Skill。例如：
  - **搜索技能：** 访问 Google/Bing 获取实时信息。
  - **计算技能：** 调用 Python 解释器或计算器处理复杂的数学问题。
  - **数据库操作：** 执行 SQL 语句查询或写入数据。
- **多模态处理类 (Multimodal)：**
  - **视觉技能：** 识别图片中的物体或 OCR 文字提取。
  - **语音技能：** 将文本转为语音（TTS）或识别语音（ASR）。
  - **图像生成：** 调用 DALL-E 或 Midjourney 生成图片。
- **特定领域业务类 (Domain-specific)：**
  - **办公自动化：** 发送邮件、创建日程、操作 Excel。
  - **代码执行：** 编写并运行代码来验证结果。
  - **金融分析：** 获取实时股价、分析财报数据。

## Skill 的组成要素

一个完整的 Skill 通常包含以下三个部分：

1.  **名称 (Name)：** 简单直观，如 `get_weather`。
2.  **描述 (Description)：** **极其重要**。大模型通过阅读这段描述来判断什么时候该用这个技能。比如：“当用户询问某个城市的天气时，调用此工具”。
3.  **参数定义 (Parameters)：** 告诉模型调用该技能需要提供哪些信息（如城市名、日期格式）。

## 举个例子

假设你有一个“旅游助手” Agent：

- **大脑：** 负责理解你的需求（“我想去上海玩两天”）。
- **Skill A (携程接口)：** 负责查询机票和酒店。
- **Skill B (高德地图)：** 负责规划景点间的交通路径。
- **Skill C (汇率转换)：** 如果你是外国人，负责把人民币价格转为美金。

**总结：** Skill 让 Agent 从一个“只能聊天”的机器人，变成了一个“能干活”的数字员工。对于开发者来说，构建 Agent 的核心工作之一就是不断为它打磨和增加高质量的 Skill。

## 为 Agent 添加自定义 Skill

在 Claude Code 中为 Agent 添加自定义 Skill（技能）主要有两种方式，具体取决于你想实现的功能复杂度。这两种方式在构建 AI 智能体应用时非常核心。

### 方式一：基于 SKILL.md 的 Agent Skills（轻量级、Prompt 与本地命令编排）

这是最简单、最原生的一种方式。你可以通过在项目中编写特定的 Markdown 文件，赋予 Claude Code 专属的上下文和执行流程。

### 方式二：基于 MCP 接入自定义工具（重量级、API 与复杂系统交互）

如果需要让 Claude Code 执行具体的代码逻辑、查询 MySQL/Redis 数据库，或者接入后端的微服务体系，就需要使用 MCP (Model Context Protocol)。MCP 是 Anthropic 推出的开源标准协议，相当于给大模型装上了一个通用的“USB-C 接口”。
