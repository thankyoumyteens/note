# 基于 SKILL.md 的 Agent Skills

### 1. 核心原理与路径放置

技能文件的本质是一个带有特殊 YAML 头信息的 Markdown 文件。存放位置决定了它的生效范围和工具兼容性。

- **项目级生效：** 放置在 `<项目根目录>/.claude/skills/<技能指令名称>/SKILL.md`
- **全局生效：** 放置在 `~/.claude/skills/<技能指令名称>/SKILL.md`

### 2. 文件的标准解剖结构

一个合格的 `SKILL.md` 必须严格包含上下两部分：YAML 头部描述和 Markdown 指令主体。

**第一部分：YAML Frontmatter (路由调度器)**

这部分必须位于文件最顶端，由 `---` 包裹。大模型完全依赖这里的 `description` 来决定何时自动触发该技能。

- `name`: 技能的唯一标识符（必须为英文/连字符，通常与文件夹名称一致）。
- `description`: 触发条件的自然语言描述。写得越像具体的“业务说明书”，AI 的自动拦截越精准。

**第二部分：Markdown 主体 (执行 SOP)**

用于定义背景上下文和分步执行指南。

- **格式警告：** 如果要在文档中向 AI 展示代码示例，**绝对禁止**使用三个反引号（` ``` `）嵌套代码块，这会破坏底层 CLI 的正则解析器。请统一改用 4 个空格的缩进形式。

### 3. 动态命令 Hook（高阶技巧）

在官方标准客户端中，Markdown 主体并不是静态死文本。你可以使用 `! 终端命令` 的语法。Agent 在阅读这份指令前，会先在 macOS 终端里静默执行该命令，并将标准的输出结果（stdout）直接注入到上下文中。

### 4. 实战案例：FastAPI 路由生成器

我们在 `~/.claude/skills/fastapi-router/SKILL.md` 中创建一个辅助 Python 后端开发的通用技能：

```markdown
---
name: fastapi-router
description: 当用户需要新建 FastAPI 接口、重构路由或编写 Pydantic 数据模型时调用此技能
---

# FastAPI 路由开发 SOP

你现在的任务是作为一个高级 Python 架构师，协助编写后端接口。

## 必须遵守的核心规范

1. 必须使用 Pydantic v2 进行请求体和响应体的数据校验。
2. 所有的路由函数必须是异步的 (async def)。
3. 必须在代码中包含清晰的 Google 风格 Docstring，明确说明输入和输出。

## 动态环境探针

当前项目中已存在的文件结构如下，请避免命名冲突：
!ls -la app/routers/

## 严格执行步骤

1. 分析用户提供的接口需求与业务逻辑。
2. 在 `app/routers/` 目录下生成对应的 `.py` 文件。
3. 自动生成一个基础的基于 `httpx` 的 `pytest` 测试用例代码片段供用户复制。
```

### 5. 如何触发与执行

配置完成后，重启终端 AI 客户端，你拥有两种唤醒它的方式：

- **隐式自动触发：** 直接在对话框中用自然语言交流（例如：“帮我加一个用户登录的验证接口”）。AI 会匹配 YAML 里的描述，自动加载 SOP。
- **显式强制触发：** 在输入框敲入斜杠加上技能名，锁定上下文。例如 `/fastapi-router 帮我写一个文件上传的路由`。
