# 如果 Codex 的计划不合格，怎么纠正

## 情况 1：它想加数据库

```text
当前计划过度设计。第 6 课只允许为内存版 POST /api/documents 制定计划。

请去掉数据库、JPA、Repository、Entity、H2、PostgreSQL 相关内容。
重新输出 7 个 Markdown 章节。
不要修改任何项目文件。
```

## 情况 2：它想改 pom.xml

```text
本课禁止新增依赖，也不需要修改 pom.xml。
请重新输出计划，并明确 pom.xml 不应修改。
```

## 情况 3：它把 content 放进成功响应

```text
成功响应设计不符合第 5 课约束。
请把成功响应收敛为只返回 documentId，不返回 title 或 content。
重新输出 API 变化和计划自检。
```

## 情况 4：它直接写代码或改文件

```text
本课是第 6 课：先计划，后执行，只允许输出计划，不允许修改代码或创建文件。

请停止修改文件。
请确认本次是否已经修改文件。
如果已修改，请说明修改了哪些文件，并给出回退建议。
然后重新只输出实现计划。
```
