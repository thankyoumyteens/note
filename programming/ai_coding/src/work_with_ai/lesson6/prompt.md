# 推荐发给 Codex 的 Prompt

在项目根目录打开 Codex，发送：

```text
目标：
为文档保存功能制定实现计划。请先不要改代码。

背景：
当前项目是 ai-doc-summary，已经完成：
1. 最小 Java 21 + Spring Boot 3.x + Maven 项目
2. AGENTS.md / CLAUDE.md 项目规则文件
3. .gitignore 和 Git baseline
4. 第 4 课的文档保存功能清晰任务
5. 第 5 课的输出格式控制

现在要为后续实现 POST /api/documents 制定可审查计划，但本课只做 Plan-Then-Act 的 Plan，不进入 Act。

第 4/5 课已确定的功能约束：
1. 接口：POST /api/documents
2. 请求 JSON：title、content
3. 成功状态码：201 Created
4. 成功响应第一版只返回 documentId
5. 参数错误返回 400 Bad Request
6. title 和 content 不能为空，也不能是纯空格
7. 当前只使用内存存储
8. 不接数据库
9. 不接 AI API
10. 不加入用户系统
11. 不加入 Spring Security
12. 不新增依赖
13. 不实现文档查询接口
14. 不实现摘要生成接口
15. 不上传文件，只接收 JSON 请求体

输入：
请查看当前项目结构、pom.xml、README.md、AGENTS.md、CLAUDE.md、src/、lessons/，以及当前 git status。

输出：
请只输出实现计划，不要修改任何项目文件。
请严格按以下 Markdown 章节输出：

## 1. 需求理解
说明本次要实现什么，以及明确不做什么。

## 2. 涉及文件
列出预计新增或修改的文件。
每个文件说明用途。
如果某个文件不应该修改，也请说明。

## 3. 修改步骤
按顺序列出实现步骤。
步骤要足够小，便于后续第 7 课小步实现。

## 4. 数据结构 / API 变化
说明请求 JSON、成功响应 JSON、错误响应 JSON、状态码和内存数据结构。
成功响应只允许包含 documentId。

## 5. 测试计划
列出需要新增或更新的测试。
至少包含：
- 保存成功
- title 为空字符串
- title 为纯空格
- content 为空字符串
- content 为纯空格
- 健康检查接口不受影响

## 6. 风险和回滚方式
说明可能风险、如何避免过度设计、如果实现失败如何通过 git 回滚。

## 7. 计划自检
用 checklist 检查：
- 是否没有数据库
- 是否没有 AI API
- 是否没有 Spring Security
- 是否没有新依赖
- 是否没有查询接口
- 是否没有摘要接口
- 是否成功响应只返回 documentId
- 是否包含测试计划
- 是否本次没有修改文件

限制：
1. 不要修改任何项目文件。
2. 不要创建新文件。
3. 不要运行命令。
4. 不要输出 Java 实现代码。
5. 不要新增依赖。
6. 不要改变第 4/5 课确定的接口约束。
7. 不要实现功能，本课只输出计划。

验收标准：
1. 输出严格包含 7 个 Markdown 章节。
2. 涉及文件列表清晰。
3. 修改步骤可以直接转化为第 7 课的小步实现任务。
4. API 设计中成功响应只包含 documentId。
5. 测试计划覆盖成功路径和参数错误路径。
6. 风险和回滚方式明确。
7. 计划自检全部符合限制。
8. Codex 确认本次没有修改任何项目文件。
```

## 你要审查 Codex 计划的 Checklist

收到 Codex 输出后，用这张表检查：

| 检查项                         | 要求     |
| ------------------------------ | -------- |
| 是否修改文件                   | 必须没有 |
| 是否有 7 个章节                | 必须有   |
| 是否列出涉及文件               | 必须有   |
| 是否要改 `pom.xml`             | 不应该   |
| 是否新增依赖                   | 不允许   |
| 是否只做 `POST /api/documents` | 必须     |
| 是否返回 `documentId`          | 必须     |
| 是否返回完整 `content`         | 不应该   |
| 是否使用内存存储               | 必须     |
| 是否有测试计划                 | 必须     |
| 是否有回滚方式                 | 必须     |
