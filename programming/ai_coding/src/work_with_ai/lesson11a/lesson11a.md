# 第 11A 课：在测试保护下小步重构

你已经完成了：

```text
第 10 课：Red，先写失败测试
第 11 课：Green，最小实现让测试通过
```

现在进入：

```text
第 11A 课：Refactor，在测试保护下小步重构
```

本课目标不是新增功能，而是检查第 11 课的 summary 实现是否有明显坏味道，并在测试保护下做**最小、安全、可回滚**的清理。

Refactor 阶段不是“必须改点什么”。

正确流程是：

```text
先评估 → 再决定是否重构 → 如果重构，只做小步 → 每步后跑测试
```

如果代码已经足够简单，可以结论为：

```text
当前 summary 实现足够小，无需重构。
```
