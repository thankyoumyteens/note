# 本课要设计的 3 个模板

## new-feature-task

用途：

```text
把一个模糊功能需求改写成清晰 AI 任务。
```

输入：

```text
功能名称
用户想做什么
当前项目状态
限制条件
```

输出：

```text
6 段式任务
接口/API 设计
测试计划
风险点
后续任务拆分
```

适用场景：

```text
新功能开始前
需求还不够清楚
还不能直接写代码
```

---

## review-diff

用途：

```text
审查当前 git diff。
```

输入：

```text
当前 git diff
相关需求或 spec
测试结果
```

输出：

```text
必须修改
建议修改
可选优化
风险点
是否需要补测试
是否违反项目规则
```

适用场景：

```text
AI 修改代码后
提交前
合并前
重构后
```

---

## write-handoff

用途：

```text
生成新会话可接手的交接文档。
```

输入：

```text
当前任务目标
已修改文件
测试结果
剩余问题
关键设计决策
```

输出：

```text
已完成什么
修改了哪些文件
还剩什么
当前阻塞点
关键设计决策
如何继续
应该运行哪些测试
```

适用场景：

```text
长会话结束前
切换工具前
切换到新 Claude Code / Codex 会话前
复杂功能暂停时
```
