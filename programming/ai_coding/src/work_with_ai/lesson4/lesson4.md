# 第 4 课：从模糊需求到清晰任务

## 本课目标

你现在已经有一个干净的 Spring Boot baseline。接下来要进入第一个业务功能：

```text
文档保存功能
```

但这一课**不是直接让 AI 写代码**。

这一课训练的是：如何把一个模糊需求：

```text
帮我做一个文档保存功能。
```

改写成 AI 能稳定执行、可审查、不会乱扩展的任务。

## 为什么这一课很重要

如果你直接对 Codex 说：

```text
帮我做一个文档保存功能。
```

它可能会自己发挥：

```text
加数据库
加 JPA
加 H2
加 PostgreSQL
加 Repository
加全局异常处理
加 Swagger
加复杂 DTO
加用户系统
加上传文件接口
加 AI 摘要接口
```

但当前课程还没到这些阶段。

现在我们只需要第一个业务闭环的**计划**：

```text
POST /api/documents
保存标题和正文
先用内存存储
不接数据库
不接 AI
不加用户系统
不加 Security
```

第 4 课的核心就是：

> **先把需求说清楚，再让 AI 计划，不要直接实现。**
