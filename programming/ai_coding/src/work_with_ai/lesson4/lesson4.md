第 1 课：最小 Spring Boot 项目

接下来进入第一个业务功能：

```text
文档保存功能
```

但第 4 课不是让 AI 直接实现功能，而是训练你如何把一句模糊需求：

```text
帮我做一个文档保存功能。
```

改写成 AI 能稳定理解、不会乱扩展、可审查的开发任务。

## 本课核心方法：6 段式任务

继续使用这个结构：

```text
目标：
背景：
输入：
输出：
限制：
验收标准：
```

这 6 个部分分别控制 AI 的行为：

| 部分     | 作用                     |
| -------- | ------------------------ |
| 目标     | 明确要做什么             |
| 背景     | 告诉 AI 当前项目状态     |
| 输入     | 告诉 AI 可以参考哪些文件 |
| 输出     | 规定 AI 本次交付什么     |
| 限制     | 防止 AI 乱扩展           |
| 验收标准 | 定义怎样算完成           |
