# 不符合预期怎么纠正

## 如果 Claude Code 直接改代码，怎么纠正

发这个：

```text
本课是第 4 课：从模糊需求到清晰任务，只要求输出任务说明和实现计划，不允许修改代码或创建文件。

请停止修改文件。
请确认本次是否已经修改任何文件。
如果已修改，请说明修改了哪些文件，并给出回退建议。

然后重新只输出：
1. 清晰任务说明
2. 需求理解
3. 接口设计
4. 建议类列表
5. 存储策略
6. 参数校验策略
7. 测试计划
8. 风险点
9. 后续任务拆分
```

## 如果计划过度设计，怎么纠正

如果它提到：

```text
JPA
Repository
Entity
H2
PostgreSQL
Spring Security
Spring AI
OpenAI API
文件上传
用户系统
```

就发：

```text
当前计划过度设计。第 4 课只需要文档保存功能的最小计划。

请重新规划，并遵守：
1. 不接数据库。
2. 不使用 JPA。
3. 不新增依赖。
4. 不实现查询接口。
5. 不实现摘要接口。
6. 不实现文件上传。
7. 不加入用户系统或 Security。
8. 不接真实 AI API 或 Spring AI。
9. 只设计 POST /api/documents。
10. 存储方案使用内存 Map。
11. 成功响应只返回 documentId。
12. 输出计划，不修改代码。
```
