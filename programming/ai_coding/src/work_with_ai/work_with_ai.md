# 指挥 AI 工作

### 第 1 阶段：从零启动项目

目标：从空目录创建最小可运行、可测试、可回滚的 Java Spring Boot 项目。

| 课程     | 主题                                   |
| -------- | -------------------------------------- |
| 第 1 课  | 把“我要一个 Java 项目”变成清晰任务     |
| 第 2 课  | 建立项目规则文件 AGENTS.md / CLAUDE.md |
| 第 3 课  | 初始化 Git 与基线提交                  |
| 第 3A 课 | 项目启动流程工具化                     |

### 第 2 阶段：基础指令能力

目标：把模糊需求变成 AI 可执行任务，并让 AI 输出结构化、可审查结果。

| 课程     | 主题                                 |
| -------- | ------------------------------------ |
| 第 4 课  | 从模糊需求到清晰任务                 |
| 第 5 课  | 输出格式控制                         |
| 第 5A 课 | Prompt 模板工具化：Commands / Skills |

### 第 3 阶段：Plan-Then-Act 与小步实现

目标：先让 AI 制定方案，再决定是否执行；实现时保持小 diff、可测试、可回滚。

| 课程     | 主题                 |
| -------- | -------------------- |
| 第 6 课  | 先计划，后执行       |
| 第 7 课  | 小步实现文档保存功能 |
| 第 7A 课 | Plan-Then-Act 工具化 |

### 第 4 阶段：Spec Workflow

目标：先手写轻量 Spec 理解本质，再使用 MCP 工具管理 spec、任务、审批和进度。

| 课程     | 主题                                    |
| -------- | --------------------------------------- |
| 第 8 课  | 轻量 Spec 结构                          |
| 第 9 课  | Spec 驱动实现文档查询功能               |
| 第 9A 课 | Spec Workflow 工具化：spec-workflow-mcp |
| 第 9B 课 | 用 spec-workflow-mcp 复做一个小功能     |

### 第 5 阶段：TDD with AI

目标：用测试驱动 AI，减少“看似正确”的代码。

| 课程      | 主题                 | TDD 阶段 |
| --------- | -------------------- | -------- |
| 第 10 课  | 先写测试，不写实现   | Red      |
| 第 11 课  | 最小实现让测试通过   | Green    |
| 第 11A 课 | 在测试保护下小步重构 | Refactor |
| 第 11B 课 | 测试工作流工具化     | Tooling  |

### 第 6 阶段：AI 工具工作流集成

目标：学习 MCP、Skills、Commands、Hooks、安全边界和项目记忆系统如何组合成稳定开发流程。

| 课程      | 主题                                                                  |
| --------- | --------------------------------------------------------------------- |
| 第 12 课  | MCP 基础与项目接入                                                    |
| 第 13 课  | 项目级 Skills / Commands 设计                                         |
| 第 13A 课 | Hooks 自动化：测试、格式化、提醒                                      |
| 第 13B 课 | 工具权限与安全边界                                                    |
| 第 13C 课 | 项目记忆系统：Decision Records / Lessons Learned / Agent Instructions |

### 第 7 阶段：AI Code Review

目标：让 AI 辅助审查 diff、兼容性、测试、安全风险和历史问题，但最终由人判断。

| 课程      | 主题                                      |
| --------- | ----------------------------------------- |
| 第 14 课  | 功能审查与回归审查                        |
| 第 14A 课 | Decision / Lesson 驱动的 Review Checklist |
| 第 15 课  | Java 后端专项审查                         |
| 第 15A 课 | Code Review 工具化                        |

### 第 8 阶段：重构、文档与长期维护

目标：把项目从“能跑”升级为“可维护、可交接、可复用”。
