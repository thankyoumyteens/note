# 第 5 课：输出格式控制

第 4 课你已经把模糊需求：

```text
帮我做一个文档保存功能。
```

改写成了清晰任务。

第 5 课要训练的是：

```text
同一个需求，如何让 AI 按不同用途、不同读者、不同格式输出。
```

也就是让 AI 不再输出一大段散文，而是输出可复制、可审查、可执行的结构化内容。

本课仍然不改代码。

## 为什么要学输出格式控制

真实开发里，你不会只让 AI 写代码。你经常需要 AI 输出不同类型的材料：

```text
给产品/业务方看的解释
给开发者看的实现计划
给接口调用方看的 JSON 示例
给测试人员看的测试清单
给 reviewer 看的审查 checklist
```

如果你不控制格式，AI 很容易输出一段混合解释：

```text
这个功能是用来保存文档的，我们可以创建一个 Controller，然后……
```

这种输出不方便复制，也不方便后续复用。

第 5 课要让 AI 按固定章节输出：

```text
1. 产品解释
2. 开发计划
3. 接口 JSON 示例
4. 测试用例清单
5. Code Review Checklist
```
