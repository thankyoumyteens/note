# 第 5 课：输出格式控制

## 本课目标

第 4 课你已经学会把模糊需求改写成清晰任务。

第 5 课进一步训练：

> **同一个需求，如何让 AI 按不同角色、不同用途、不同格式输出结果。**

也就是说，不再只是让 AI “回答一下”，而是明确要求它输出：

```text
产品解释
开发计划
接口 JSON
测试清单
代码审查 checklist
```

这对以后用 Codex / Claude Code 很重要，因为真实开发中你经常需要 AI 产出不同类型的材料：

```text
给产品经理看的解释
给开发者看的实现计划
给测试用的测试清单
给接口调用方看的 JSON 示例
给 reviewer 用的审查清单
```

## 为什么输出格式控制很重要

如果你只说：

```text
帮我整理一下文档保存功能。
```

AI 很可能输出一大段散文，既不好复制，也不好执行。

更好的方式是明确：

```text
请按 Markdown 标题分区。
JSON 示例放在代码块中。
测试清单用 checkbox。
review checklist 按风险类别分组。
不要修改代码。
```

这就是输出格式控制。
