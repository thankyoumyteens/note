# 本课任务结构

以后你指挥 AI 创建项目，默认使用 6 段式：

```text
目标：
背景：
输入：
输出：
限制：
验收标准：
```

这 6 段分别解决不同问题。

## 目标：告诉 AI 做什么

目标必须具体。

差：

```text
创建一个 Java 项目。
```

好：

```text
从空目录创建一个最小可运行的 Java 21 + Spring Boot 3.x + Maven 项目，项目名为 ai-doc-summary。
```

这里明确了：

```text
Java 版本
Spring Boot 版本范围
构建工具
项目名
当前状态：空目录
项目复杂度：最小可运行
```

---

## 背景：告诉 AI 为什么这样做

背景不是废话，它会影响 AI 的取舍。

```text
背景：
这是 AI 工作指挥方法课程的实战项目。当前目录为空，没有任何已有代码。项目后续会逐步实现文档保存、AI 摘要生成、摘要查询等功能。
```

这段话告诉 AI：

```text
现在只是课程起点
后面会扩展
当前不要一次性做完
项目方向是 AI 文档总结
```

---

## 输入：告诉 AI 当前有什么

从零项目的输入很简单：

```text
输入：
当前只有空目录。你可以创建 Maven 项目结构和必要文件。
```

这句话很重要，因为它明确告诉 AI：

```text
不要假设已有代码
不要去修不存在的文件
可以创建新文件
```

---

## 输出：告诉 AI 要交付什么

输出要具体到文件级别。

```text
输出：
请创建最小可运行项目，包括：
1. pom.xml
2. 主启动类
3. 一个健康检查接口 GET /api/health
4. 一个基础测试
5. README.md 中写明启动和测试命令
```

这里不让 AI 自由发挥，而是指定第一版项目必须有的最小文件。

---

## 限制：告诉 AI 不要做什么

限制是本课最关键的部分。

```text
限制：
1. 不要加入数据库。
2. 不要加入 Spring Security。
3. 不要加入 Docker。
4. 不要加入前端。
5. 不要接入真实 AI API。
6. 不要生成复杂分层，只保留最小结构。
```

这可以防止 AI 一上来把项目做重。

尤其注意这几条：

```text
不要数据库
不要 Security
不要 Docker
不要真实 AI API
```

这些都是后续课程会慢慢加入或讨论的内容，不应该在第 1 步出现。

---

## 验收标准：告诉 AI 怎样算完成

验收标准必须能检查。

```text
验收标准：
1. mvn test 可以通过。
2. mvn spring-boot:run 可以启动。
3. GET /api/health 返回 OK 或类似健康状态。
4. 项目结构清晰，文件数量尽量少。
5. README.md 包含运行方式。
```

这比“项目要能跑”强得多。

因为你拿到 AI 结果后，可以直接检查：

```bash
mvn test
mvn spring-boot:run
curl http://localhost:8080/api/health
```
