# 第三步：运行测试

实现后，让 Codex 或你自己运行：

```bash
mvn test
```

如果让 Codex 执行，可以发：

```text
目标：
继续第 9 课。现在运行测试并报告结果。

请运行：
mvn test

要求：
1. 如果测试通过，请总结结果。
2. 如果测试失败，请先分析失败原因。
3. 修复失败时只做最小修改。
4. 不要新增依赖。
5. 不要改变 POST /api/documents 成功响应格式。
6. 不要修改无关文件。
7. 修复完成后再次运行 mvn test。
```

如果测试失败，不要只发“报错了”，要把完整错误日志贴给 Codex。
