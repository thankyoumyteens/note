# spec-workflow-mcp 是什么

`spec-workflow-mcp` 是一个帮助 AI 按步骤开发项目的工具。

它的作用是：**不要让 AI 一上来就直接写代码，而是先把需求、设计、任务拆清楚，再开始实现。**

这样可以减少 AI 乱改代码、误解需求、扩大范围的问题。

## 为什么需要它

如果直接对 AI 说：

```text
帮我加一个功能
```

AI 可能会出现这些问题：

- 没有完全理解需求
- 自己脑补功能
- 改了不该改的文件
- 一次改太多代码
- 忘记项目原来的规则
- 写出的代码和项目风格不一致
- 出问题后很难回滚

`spec-workflow-mcp` 的目的就是给 AI 加一套开发流程，让它按规矩做事。

## 它的基本流程

通常分成几个阶段：

### 1. 项目上下文

先让 AI 了解项目：

- 这个项目是做什么的
- 用了什么技术栈
- 代码目录怎么组织
- 哪些东西不能随便改
- 后续开发要遵守什么规则

这些内容通常放在：

```text
.spec-workflow/steering/
```

### 2. 需求文档

把要做的功能写清楚。

重点说明：

- 要做什么
- 不做什么
- 输入是什么
- 输出是什么
- 出错时怎么处理
- 怎么判断功能完成

这一阶段只写需求，不写实现方案。

### 3. 设计文档

在需求确认后，再讨论怎么实现。

重点说明：

- 需要改哪些模块
- API 怎么变化
- 数据如何保存
- 错误如何处理
- 需要哪些测试
- 哪些技术不能引入

这一阶段只做设计，不直接写代码。

### 4. 任务拆分

把设计拆成一个个小任务。

每个任务都要做到：

- 范围小
- 可测试
- 可回滚
- 只解决一个问题
- 完成后停止，等待检查

### 5. 执行和审批

AI 不能自己一路做到底。

你需要在关键阶段检查并确认：

- 需求是否正确
- 设计是否合理
- 任务是否拆得太大
- 代码是否越界修改
- 测试是否通过

只有确认后，AI 才继续下一步。

## 简单理解

`spec-workflow-mcp` 就是给 AI 编程加一套流程：

1. 先理解项目
2. 再确认需求
3. 再做设计
4. 再拆任务
5. 最后小步写代码

它不是为了让开发变复杂，而是为了让 AI 写代码更可控。
