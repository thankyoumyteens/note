# 验收文档

第 3 轮不要直接让 AI 说“验收通过”。你自己按下面模板检查，然后把结论发给 AI。

```
我正在进行第 3 轮人工验收。

请不要代替我验收，也不要自我声明文档可信或通过。

我会从以下角度亲自检查阶段 0 文档草稿：

1. 可信性检查

   * 每个结论是否能从项目文件中找到依据
   * 是否把“推测”写成了“事实”
   * 是否把“未确认事项”写成了确定结论

2. 脑补检查

   * 是否编造了项目目标
   * 是否编造了 Java 版本
   * 是否编造了架构设计意图
   * 是否编造了测试策略
   * 是否编造了不存在的工程规范

3. 越界检查

   * 是否提出了本阶段不该执行的重构
   * 是否试图修改业务代码
   * 是否试图修改 pom.xml
   * 是否试图新增依赖
   * 是否把后续功能需求提前设计了
   * 是否把 `.spec-workflow/` 直接当成正式仓库规则

4. 文档放置检查

   * product.md 是否只写项目目标和业务边界
   * tech.md 是否只写技术栈、构建、测试和约束
   * structure.md 是否只写目录结构和代码组织
   * docs/lessons/README.md 是否只写 lessons 目录规则

我的人工验收结论如下：

* 可信性：通过 / 需要修改
* 脑补问题：无 / 有，具体如下：
* 越界问题：无 / 有，具体如下：
* 需要修改的地方：
  1.
  2.
  3.

请根据我的验收意见，只修改阶段 0 文档草稿。

严格限制：

* 不要写入文件，除非我明确要求。
* 不要修改业务代码。
* 不要新增依赖。
* 不要进入阶段 1。
* 修改草稿后停止，等待我再次确认。
```
