# 要实现的需求

```markdown
# 宠物就诊预约管理需求草稿

## 1. 需求目标

为 `spring-petclinic-rest` 增加宠物就诊预约管理能力，使客户端可以为指定宠物预约指定兽医的就诊时间，并支持查询和取消预约。

该功能用于区分“未来预约”和“已经发生的就诊记录”，避免系统只能记录 visit，而无法表达预约流程。

## 2. 核心场景

用户需要为某只宠物创建一次就诊预约。

预约时需要指定：

- 宠物
- 兽医
- 预约时间
- 预约原因

系统需要检查预约是否合理，例如宠物和兽医是否存在、预约时间是否冲突。

创建成功后，客户端可以查询预约信息；如果预约不再需要，也可以取消预约。

## 3. 功能范围

本次需求包含：

1. 创建宠物就诊预约
2. 查询某只宠物的预约列表
3. 查询某个兽医某天的预约列表
4. 取消预约
5. 防止同一兽医在同一时间被重复预约
6. 防止同一宠物在同一时间被重复预约

## 4. 基本业务规则

1. 每个预约必须绑定一个已存在的宠物。
2. 每个预约必须绑定一个已存在的兽医。
3. 每个预约必须包含预约时间。
4. 每个预约必须包含预约原因。
5. 同一兽医在同一时间只能有一个有效预约。
6. 同一宠物在同一时间只能有一个有效预约。
7. 预约状态至少包含：
   - `SCHEDULED`
   - `CANCELLED`

8. 取消预约不删除数据，只将状态改为 `CANCELLED`。
9. 已取消的预约不参与时间冲突判断。

## 5. 不做事项

本次需求不做：

1. 不做支付。
2. 不做短信、邮件或 App 通知。
3. 不做复杂排班系统。
4. 不做权限系统。
5. 不做预约改期。
6. 不做预约完成后的自动 visit 生成。
7. 不修改现有 visit 的核心行为。
8. 不引入外部依赖。
9. 不引入消息队列、缓存、搜索引擎等新技术。

## 6. 错误处理

需要处理以下错误情况：

1. 宠物不存在。
2. 兽医不存在。
3. 预约时间为空或格式非法。
4. 预约原因为空。
5. 兽医在该时间已有有效预约。
6. 宠物在该时间已有有效预约。
7. 预约不存在。
8. 重复取消已经取消的预约。

## 7. 验收标准

需求完成后，应满足：

1. 可以成功创建一条有效预约。
2. 创建预约时会校验宠物和兽医是否存在。
3. 同一兽医同一时间不能被重复预约。
4. 同一宠物同一时间不能被重复预约。
5. 可以查询某只宠物的预约列表。
6. 可以查询某个兽医某天的预约列表。
7. 可以取消预约。
8. 取消后的预约不会被物理删除。
9. 取消后的预约不再阻止同一时间再次预约。
10. 现有 owner、pet、vet、visit 接口行为不被破坏。
```
