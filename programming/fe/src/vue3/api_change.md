# Options API 和 Composition API

Options API 是 Vue 2 中的传统方式来定义组件的行为。它是基于选项(如 `data`, `methods`, `computed`, `watch` 等)的对象结构来组织代码的。每个选项都有特定的作用域和生命周期钩子。

主要特点包括：

- **易于理解**：对初学者友好，因为每个选项都清楚地对应着组件的一个特定方面。
- **简单的状态管理**：通过 `data` 对象定义组件的状态。
- **明确的生命周期钩子**：提供了一系列明确的生命周期钩子方法，如 `created`, `mounted`, `updated` 等。

Composition API 是 Vue 3 引入的一种新的方式来组织和重用组件逻辑。它的设计目的是为了解决在使用 Vue 2 的 Options API 时遇到的一些局限性，尤其是在处理复杂组件时的状态管理和逻辑复用方面的问题。通过 Composition API，可以将相关功能的代码组织在一起，这使得代码更加模块化、可读性和可维护性更强。

主要特点包括：

- **基于函数**：Composition API 主要是围绕 `setup` 函数构建的。
- **更好的逻辑复用**：可以很容易地提取出一个或多个逻辑相关的函数，并在不同的组件间共享这些函数。
- **更灵活的响应式系统**：提供了 `ref` 和 `reactive` 来创建响应式数据。
- **更好的类型推断**：对于 TypeScript 用户来说，Composition API 提供了更好的类型支持。
